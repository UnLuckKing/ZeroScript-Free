# ZeroScript 1.30 — Superior Engine

## Added

- Deterministic **Intent Compiler** that classifies tasks, scores risk, selects phases, recommends providers and explains every local decision.
- **Project Genome** scanner for scripts, remotes, ScreenGuis, inferred systems and dependency edges.
- **Shadow Guard** scoped sandbox snapshots under `ServerStorage.ZeroScriptShadow` for high-risk work.
- **Proof Engine** with verified/unverified status, evidence checks and confidence score.
- Persistent per-project **Design DNA** and reusable **Behavioral Contracts**.
- Studio **Command Palette** that attaches selected Explorer paths and can be bound to Ctrl+K through Studio shortcuts.
- **Model Jury** requirement for high/critical tasks using independent reviewer comparison.
- Opt-in **Self-Healing** modes: off, suggest and auto-shadow.
- Local **Live Game Brain** for deterministic JSON/CSV Roblox Analytics recommendations.
- New modern Hub **Üstün Sistem** screen for all Superior controls.

## How decisions work without a built-in AI

ZeroScript does not contain a hidden language model. Local deterministic rules use task keywords, risk thresholds, Project Genome counts, Recipe scores, Design DNA and recorded provider outcomes to decide scope, safety, routing and required tests. Connected browser AIs still perform creative implementation and diagnosis.

## Safety

- Stored memory and Project Genome remain guidance, never proof.
- Shadow Guard is a scoped transaction/sandbox layer inside the current Studio connection, not an independent second Studio process.
- Self-Healing defaults to suggestion mode and never silently publishes changes.
- Proof Engine refuses verified status when required playtest, Output, visual or jury evidence is missing.
- Existing checkpoints, rollback, writer locks, permission scopes, catastrophic-change blocking and server-authoritative rules remain active.

## Validation limits

The repository includes Python compile checks, Superior Engine unit tests, control API tests, JavaScript syntax checks and deterministic Superior Pack tests. Real provider websites, Roblox Studio plugin behavior, screenshots, multi-client sessions and production DataStores still require live Windows testing.
